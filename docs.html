API 文档

完整接口参考，快速接入星算纪 AI 聚合服务

🚀 快速开始

1. 获取 API Key

注册并登录后，在用户中心 → API 密钥页面创建您的 API Key。请妥善保管，不要在前端代码中暴露。

2. 安装 SDK（可选）

# Node.js
npm install @xingapis/sdk

# Python
pip install xingapis-sdk

3. 发起第一个请求

curl https://api.xinsuanji.cn/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

💡 提示：我们的 API 接口与 OpenAI 完全兼容，如果您已经使用 OpenAI SDK，只需更改 base URL 和 API Key 即可。

🔐 认证方式

所有 API 请求都需要在 HTTP Header 中携带您的 API Key：

Authorization: Bearer YOUR_API_KEY

⚠️ 警告：请勿在前端代码（浏览器、移动端 App）中硬编码 API Key！请在后端服务器中使用，或通过我们的代理服务。

💬 对话补全 (Chat Completions)

这是最核心的接口，用于与 AI 模型进行对话。

HTTP 请求

POST /v1/chat/completions
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY

{
  "model": "gpt-4",
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "解释量子计算"}
  ],
  "temperature": 0.7,
  "max_tokens": 1000,
  "stream": false
}

请求参数

参数名 类型 必填 说明
model string 必填 模型名称，如 gpt-4, claude-3-5-sonnet, gemini-pro
messages array 必填 对话消息数组，每个对象包含 role (system/user/assistant) 和 content
temperature number 可选 采样温度，0-2 之间，默认 0.7。越高越随机，越低越确定
max_tokens integer 可选 最大生成 token 数，默认 1000
stream boolean 可选 是否启用流式响应，默认 false

❌ 错误码说明

HTTP 状态码 错误码 说明
400 invalid_request_error 请求参数错误，请检查参数格式和类型
401 authentication_error API Key 无效或已过期
429 rate_limit_error 超出频率限制，请稍后重试
500 server_error 服务器内部错误，请联系技术支持
